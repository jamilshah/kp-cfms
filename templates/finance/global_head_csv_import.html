5-Level CoA Structure

This import creates the complete hierarchy:

Level 1 Classification: A, C, G, K
Level 2 Major Head: A01, C03 (3 chars)
Level 3 Minor Head: A011, C038 (4 chars)
Level 4 NAM Head: A01101, C03880 (5+ chars)
Level 5 Sub-Head: A01101-01 (optional)
How It Works
1. Pre-Validation

All data is validated before any database changes. Errors are reported with row numbers.

2. Atomic Import

If validation passes, all levels are created in one transaction. If anything fails, nothing is imported.

3. Automatic Hierarchy

Parent records (Major → Minor → NAM) are automatically created or linked based on codes.

CSV Rules
  • Code Hierarchy: Minor must start with Major, NAM must start with Minor
  • Sub-Heads: Optional. If provided, parent NAM head cannot be posted directly
  • Account Type: EXP, REV, AST, LIA, or EQT
  • Scope: UNIVERSAL (all depts) or RESTRICTED (specific depts)
  • Duplicates: Same code with different details will be rejected
  • Allow Direct Posting: Must be N when Sub-Heads exist
